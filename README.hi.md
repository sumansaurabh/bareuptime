## 🤔 BareUptime क्यों?

पारंपरिक अपटाइम मॉनिटरिंग टूल्स **अत्यधिक महंगे** हैं। हमें वास्तव में केवल एक ऐसी प्रणाली की आवश्यकता है जो दो सरल प्रश्नों का उत्तर देती है:

1. **क्या मेरी वेबसाइट/API चालू है?**
2. **क्या यह तुरंत सूचित कर सकती है** — मोबाइल, Slack, Discord, या ईमेल पर?

बस इतना ही। हमें फैंसी एनिमेटेड डैशबोर्ड या एंटरप्राइज सेल्स कॉल की आवश्यकता नहीं है।

## 🛠️ टेक स्टैक

- **फ्रंटएंड**: Next.js 15, TypeScript, Tailwind CSS
- **UI कंपोनेंट**: Radix UI, shadcn/ui
- **डेटाबेस**: Supabase (PostgreSQL)
- **मॉनिटरिंग**: ग्लोबल वर्कर पूल
- **नोटिफिकेशन**: Push, Email, Webhooks, Slack, Discord
- **मोबाइल ऐप्स**: React Native (iOS और Android)

## 🧑‍💻 योगदान

हम योगदान का स्वागत करते हैं! यह प्रोजेक्ट उन डेवलपर्स द्वारा बनाया गया है जो अधिक कीमत वाले मॉनिटरिंग टूल्स से थक गए थे।

### लोकल डेवलपमेंट

1. **रिपॉजिटरी क्लोन करें**
   ```bash
   git clone https://github.com/sumansaurabh/bareuptime.git
   cd bareuptime
   ```

2. **डिपेंडेंसी इंस्टॉल करें**
   ```bash
   npm install
   # या
   pnpm install
   ```

3. **एनवायरनमेंट सेटअप**
   ```bash
   cp .env.example .env.local
   # अपनी Supabase और अन्य API keys जोड़ें
   ```

4. **डेवलपमेंट सर्वर चलाएं**
   ```bash
   npm run dev
   # या
   pnpm dev
   ```

5. **ब्राउज़र खोलें**
   [http://localhost:3000](http://localhost:3000) पर नेविगेट करें

### प्रोजेक्ट संरचना
```
bareuptime/
├── app/                 # Next.js ऐप राउटर
│   ├── api/            # API रूट्स
│   ├── components/     # पेज-विशिष्ट कंपोनेंट
│   └── globals.css     # ग्लोबल स्टाइल
├── components/         # शेयर्ड UI कंपोनेंट
│   └── ui/            # shadcn/ui कंपोनेंट
├── hooks/             # कस्टम React hooks
├── lib/               # यूटिलिटी फंक्शन
└── public/            # स्टैटिक assets
```
